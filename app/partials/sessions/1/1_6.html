Creating Variables
  • Declared Variables
    Variables can be declated with var keyword.
  • Undeclared Variables
    Its not mandatory to declare them, though in ECMAScript5 under strict mode an error is thrown for undeclared variables.
  • Repeated declaration
    It is legal and harmless to declare a variable more than once with the var statement. If the repeated declaration has an initializer, it acts as if it were simply an assignment statement.
Variable scopes
  • Undeclared variables
    undeclared variables under global scope.
  • Declared variables
    Declared variables have local scopes within the region they are declared.
    • Global Variables
      A variable declared in the global region (i.e outside any function) has a global scope.
    • Function socpe
      Varibales in JavaScript do not support block scopes, instead they have function scopes.
      It does not matter where you define your variable in a function they are vissible through out the function. All parameters to function have the same scope.
      A function scope variable hides the same named global variable.
    • Scope chain / Closure scope
      This scope chain is a list or chain of objects that defines the variables that are “in scope” for that code.
      In top-level JavaScript code , the scope chain consists of a single object, the global object. In a non-nested function, the scope chain consists of two objects. The first is the object that defines the function’s parameters and local variables, and the second is the global object. In a nested function, the scope chain has three or more objects.
      When a function is defined, it stores the scope chain then in effect. When that function is invoked, it creates a new object to store its local variables, and adds that new object to the stored scope chain to create a new, longer, chain that represents the scope for that function invocation.
